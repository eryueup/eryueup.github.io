自动部署到GitHub Pages

网上关于部署的方法也很多，但是自己部署时总是会遇到一些莫名奇妙的问题：总的来说就是Github Actions的CI流程错误导致的

在实现自动部署之前，需要实现hexo本地安装并且可以正确访问，不需要实现远端部署，因为直接通过Git推送

hexo和主题配置

hexo有许多主题,也可以在GitHub上直接搜索，此处配置的是keep主题官网

hexo和keep主题的_config.yml文件跟着官方修改为适合自己的就可以了，官方文档此处也是比较详细的。

GitHub配置

keep官方自动部署配置

简要：使用远端私有仓库hexo-source存储hexo源文件，公开仓库xxx.github.io存储hexo生成的public的资源文件，也就是将来访问的站点资源，其中xxx为你的GitHub用户名；另外需要生成一个Token令牌,需要赋予此令牌repo和workflow权限：此令牌作用就是配置到私有仓库hexo-source的Secrets赋予其权限，用于执行workflow下的CI流程，并将生成的public资源文件部署到公开仓库xxx.github.io；这一部分需要更改.github\workflows\hexo-deploy.yml的相关配置

此处使用的是全局Token令牌，权限比较大，这篇文章配置的是两个仓库之间的也比较适合利用 Github Actions 自动部署 Hexo 博客

此处使用的是两个仓库实现自动部署，也可以一个仓库利用分支：我反正没成功，无所谓，就是玩。

Git配置

在完成上述Github配置后，在本地hexo主目录下与远端私有仓库hexo-source建立连接，推送到远端实现自动化。

问题及解决

图片的插入

资源文件夹

官方有说明图片的管理，所以图片较少时无需图床，配合Typora编辑很舒服

我使用的是第一种方法：建立source/images文件夹，将文章图片放进去，在文章中即可通过`images/xxx.png引用图片

hexo在解析时，会自动识别此处图片

对于每个文章一个文件夹我觉得难以管理，主要是我之前的文章图片都是存储在一个文件夹，不好迁移所以没有使用

在本地阅读时，只需在Typora中设置图像根目录即可，将其设置为source即可

Hexo 搭建个人博客（五）Hexo 资源文件夹

Hexo博客写作与图片处理的经验

Hexo 引用本地图片以及引用本地任意位置图片的一点思路

图片的插入应该注意分隔符：每个平台与语言都是有差异的，解析会导致错误

图表的插入

此处暂未使用过，可以使用js的插件：echarts，在文章中直接调用，实现图表显示

生成的index.html内容为空

出现这种问题，可以正常访问站点，但是却得不到内容，公开仓库xxx.github.io的index.html为空，网上说是nodejs版本过低，使用12+可以解决，我当时使用的是14也出现此问题，后来玄学解决：冷静后再次部署

标题、标签、分类无法使用空格

在front matter的内容尽量避免使用空格#等html需要转义的符号，会导致展示异常，如下C#被解析为C，我改了CSharp

更严重的是使用前端的标签，会导致解析错误，使用时可以引用如：script

发布时隔undefined

系统时区与生成站点使用的时区不一致，导致时间差非法，如我的系统是日文，我使用中文生成，会出现此问题

无法实现多种分类

hexo官网说明了，分类尽可能明确，只允许有一个父类，其他都是子类

访问速度较慢

可以使用CDN加速

keep主题配置后不生效

因为我使用了平滑升级，所以在推送时keep.yml覆盖了_config.yml，因此在推送前我将_config.yml复制到了keep.yml中便可以解决。

不展示markdown文件

根据hexo官方说明，在source文件夹下建立_xxx类似文件即可，会自动忽略此文件夹下面的内容，所以我将个人的学习笔记放在了这样的目录下，以便同步笔记，或许也可以使用笔记加密的方法(没有了解过)

GitTalk错误

第一次加载后 Error: Bad credentials,可以登陆，登陆后加载 Error: no access token，这个错误是使用了eryueup.github.io这个已经存在的仓库建立的issue,重新按照步骤新建仓库解决。

Error: Network Error网上大多说网络问题。

其他的问题可以查看官方issue
